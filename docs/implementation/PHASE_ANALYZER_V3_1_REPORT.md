# Phase Analyzer V3.1 Report: New Analyzer Contract And Shadow Parser

Date: 2026-06-06

## Summary

Phase Analyzer V3.1 is complete.

Implemented Analyzer Contract V3 as a compact observation-only schema and added a non-authoritative shadow parser before the analyzer node. The current planner, memory, stage, objection, and response services still consume `TurnAnalysis`, so Phase 1 includes a temporary compatibility adapter that derives legacy runtime aliases from the V3 observation. The LLM schema no longer asks for those legacy fields.

No evaluation datasets were modified. No migrations were created or applied. No full replay was run.

## Implemented

- Added `analyzer_contract_version = "3.0"`.
- Added closed enums for requested capabilities, requested actions, requested products, and V3 prior-reference types.
- Added `AnalyzerTurnObservation` with:
  - independent linguistic observations;
  - extracted facts;
  - requested capabilities;
  - requested external actions;
  - requested product;
  - objection candidate;
  - purchase readiness;
  - prior reference;
  - post-purchase signal;
  - confidence.
- Added evidence validation for required spans.
- Added OpenAI structured-output schema for Analyzer Contract V3.
- Updated analyzer prompt to forbid dependent commercial fields.
- Added temporary V3-to-`TurnAnalysis` adapter for Phase 1 compatibility.
- Added shadow parser node:

```text
load_memory
→ parse_explicit_signals_shadow
→ analyze_turn
```

- Added parser trace and analyzer observation to `response_metadata`.
- Documented the contract in Markdown and JSON.

## Compatibility Note

The LLM no longer generates dependent fields such as:

- `has_objection`
- `references_prior_message`
- `explicit_turn_number`
- `explicit_start_intent`
- `action_requirement`
- `known_product_fit`
- `recommended_product`
- `macro_action`
- `micro_action`
- `cta_type`
- `needs_rag`

During Phase 1, legacy runtime aliases are derived in code from the V3 observation so existing planner and response behavior can continue. Phase 2 should replace this adapter with a deterministic normalized-turn contract.
