# 05 - Casos de uso y segmentación por producto

Este documento ayuda al agente de preventa a recomendar el producto correcto según el tipo de necesidad.

## Regla general

La diferencia principal entre MovIA Captura y MovIA Híbrido es:

- MovIA Captura responde y captura información.
- MovIA Híbrido responde, captura y ejecuta hasta 2 acciones simples.

## MovIA Captura: casos ideales

MovIA Captura funciona muy bien para negocios que reciben muchos mensajes y necesitan responder información.

### Ejemplos de negocios

- Dentistas
- Médicos
- Barberías
- Estéticas
- Servicios profesionales
- Oficios
- Fumigación
- Carpintería
- Locales de impresión
- Tiendas pequeñas
- Negocios con campañas de Facebook
- Negocios que reciben mensajes por WhatsApp Ads
- Negocios con preguntas frecuentes repetidas

### Problemas que resuelve

- Tardan mucho en responder.
- Pierden leads por no contestar rápido.
- Reciben las mismas preguntas todos los días.
- Necesitan filtrar interesados reales.
- Quieren redirigir leads a una persona humana.
- Quieren ordenar la información antes de atender manualmente.

### Ejemplos de preguntas que puede responder

- ¿Cuánto cuesta?
- ¿Dónde están ubicados?
- ¿Qué servicios ofrecen?
- ¿Cuáles son los horarios?
- ¿Qué requisitos necesito?
- ¿Tienen disponibilidad?
- ¿Cómo funciona?
- ¿Qué incluye?
- ¿Me pueden dar más información?

### Cuándo NO recomendar MovIA Captura

No recomendar MovIA Captura cuando el cliente necesita:

- Agendar citas.
- Cotizar automáticamente.
- Conectarse con paneles.
- Revisar tickets.
- Registrar garantías.
- Consultar proveedores.
- Acciones dentro de sistemas externos.
- Automatización operativa.
- Seguimiento estructurado del lead.
- Procesos de venta persuasivos.

En esos casos, recomendar MovIA Híbrido.

## MovIA Híbrido: casos ideales

MovIA Híbrido funciona cuando el cliente necesita que el agente haga algo además de responder.

### Ejemplos de uso

- Agendar citas.
- Cotizar servicios.
- Dar seguimiento al lead.
- Enviar recordatorios.
- Guiar a usuarios por un proceso.
- Capturar datos específicos.
- Registrar solicitudes.
- Procesar información recibida por WhatsApp.
- Organizar tickets, fotos o datos enviados por clientes/proveedores.

### Ejemplo concreto

Un cliente tiene 5 proveedores. Los proveedores envían tickets de compra con una foto del detalle. A veces mandan datos diferentes para garantía.

Este caso no corresponde a MovIA Captura, porque no se trata solo de responder preguntas. Requiere procesar información, posiblemente conectarse a un panel o sistema, organizar datos y responder con base en un proceso.

Este caso debe ir a MovIA Híbrido o cotización especial, según complejidad.

## MovIA Ventas: casos ideales

Aún no disponible.

Será ideal para negocios que necesitan que el agente influya en la decisión de compra.

Ejemplos:

- Negocios con leads que comparan opciones.
- Servicios donde el cliente tiene objeciones.
- Productos que requieren explicación de valor.
- Negocios donde responder dudas no es suficiente.
- Casos donde el agente debe guiar hacia compra o agenda.

## MovIA Pro Comercial: casos ideales

Aún no disponible.

Será ideal para negocios que requieren un agente comercial avanzado, con mayor memoria, más personalización y lógica conversacional más compleja.

Debe manejarse como solución avanzada y potencialmente como consultoría.
