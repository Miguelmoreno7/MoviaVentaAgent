# 03 - Políticas comerciales y operativas de MovIA

Este documento contiene las políticas que el agente de preventa debe usar como fuente oficial.

## Depósito inicial

Para comenzar un proyecto, el cliente debe pagar un depósito equivalente al 50% del valor total del setup.

Sin este depósito, MovIA no realizará trabajo personalizado para la empresa.

El depósito confirma que el cliente desea iniciar el proyecto y permite que MovIA comience la construcción del agente.

## Pago restante

El 50% restante se paga después de que el agente haya sido construido y aprobado por el cliente.

El cliente podrá revisar pruebas, escenarios y resultados antes de aprobar.

No se solicita el pago restante antes de la aprobación del agente.

## Aprobación del agente

Antes de activar el agente en el número de WhatsApp del cliente, MovIA entrega evidencia del comportamiento del agente en ambiente de pruebas.

El cliente puede:

- Aprobar el agente.
- Solicitar ajustes dentro del alcance contratado.

Cuando el cliente aprueba el agente, se habilita el pago final.

## Inicio de mensualidad

La mensualidad empieza a correr desde el momento en que el cliente aprueba el agente.

La mensualidad cubre mantenimiento, operación, infraestructura, recursos computacionales y soporte dentro del alcance.

## Política de no reembolso

No hay reembolsos por ningún motivo.

Esto incluye el depósito inicial.

El cliente debe revisar las políticas y entender el proceso antes de pagar.

El depósito cubre el inicio del trabajo personalizado y la dedicación de recursos por parte de MovIA.

## Falta de pago mensual

Si el cliente no paga la mensualidad, el agente se pausa.

Cuando el cliente vuelva a pagar la mensualidad correspondiente, el agente podrá volver a activarse.

## Soporte incluido

Una vez que el cliente paga el depósito, recibe acceso a atención más personalizada con Miguel por WhatsApp.

Este acceso se ofrece para dar seguimiento cercano durante la primera fase del proyecto.

El soporte incluido depende del plan contratado:

- MovIA Captura: hasta 2 juntas de 1 hora antes del release y hasta 2 juntas al mes.
- MovIA Híbrido: hasta 2 juntas de 1 hora antes del release y hasta 2 juntas al mes.
- MovIA Ventas: hasta 3 juntas antes del release y hasta 3 juntas al mes.
- MovIA Pro Comercial: pendiente de definir formalmente, pero debe manejarse como solución avanzada.

Las juntas incluidas no son ilimitadas.

MovIA no puede agendar llamadas ilimitadas porque se atienden otros clientes y proyectos.

## Qué tipo de soporte se incluye

El soporte mensual incluye:

- Actualización de información del negocio.
- Ajustes de respuestas.
- Correcciones de comportamiento dentro del alcance.
- Ajustes de base de conocimiento.
- Actualización de precios, servicios, catálogos o FAQs.
- Revisión de cómo está respondiendo el agente.
- Mejoras menores de tono o instrucciones.

## Qué tipo de soporte NO se incluye

La mensualidad no incluye:

- Nuevas características.
- Nuevas acciones no contratadas.
- Integraciones adicionales fuera del alcance.
- Cambiar de MovIA Captura a MovIA Híbrido sin cotización.
- Agregar agendamiento si el cliente contrató MovIA Captura.
- Agregar cotizaciones si no estaban incluidas.
- Desarrollar features específicos sin cotización.
- Consultoría ilimitada.
- Servicio postventa complejo.
- Soporte 24/7 humano.

## Cambios al agente

Los cambios dentro del mismo alcance no se cobran aparte.

Ejemplos de cambios incluidos:

- Actualizar información.
- Actualizar precios.
- Cambiar respuestas.
- Agregar información nueva a la base de conocimiento.
- Corregir instrucciones.
- Ajustar el tono del agente.
- Modificar cómo responde ante ciertos casos.

Los cambios que incrementen el alcance sí se cobran aparte.

Ejemplos de cambios que pueden requerir cotización:

- Agregar agendamiento.
- Agregar cotización automática.
- Conectar con un CRM.
- Conectar con sistemas externos.
- Agregar nuevas acciones.
- Automatizar procesos nuevos.
- Crear funcionalidades específicas.
- Cambiar a un agente más avanzado.

## Tokens y uso de API

El costo de tokens/API viene incluido dentro de la mensualidad bajo un uso razonable.

Si un cliente excede significativamente el límite esperado de uso, MovIA le notificará que puede requerirse un ajuste o cargo adicional mensual.

La intención no es cobrar extra en casos normales, sino proteger la operación cuando el volumen sea mucho mayor al considerado.

## Canales

Actualmente MovIA soporta WhatsApp.

Facebook e Instagram están en proceso para ofrecerse más adelante.

## Conexión oficial con Meta

MovIA utiliza la conexión oficial de Meta/Facebook para WhatsApp Business.

No se utilizan plataformas no oficiales para conectar números de WhatsApp.

Esto implica que el cliente necesita:

- WhatsApp Business.
- Cuenta de Facebook.
- Permisos suficientes sobre el activo.
- Capacidad de completar el registro oficial.

Si el cliente tiene problemas con la conexión de canales, debe comunicarse directamente con Miguel para recibir orientación.
