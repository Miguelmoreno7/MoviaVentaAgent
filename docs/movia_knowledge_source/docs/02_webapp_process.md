# 02 - Proceso de la web app de MovIA

Link oficial de la app:

https://app.moviatech.com.mx

Este documento describe el proceso que debe explicar el agente de preventa cuando un cliente pregunte cómo iniciar, cómo registrarse o cómo funciona la plataforma.

## 1. Registro en la aplicación

El cliente debe entrar a la app de MovIA y registrarse.

La aplicación puede cambiarse entre inglés y español. El cliente puede seleccionar el idioma que prefiera.

## 2. Selección de workspace

Después de registrarse, la plataforma redirige al usuario a una pantalla donde puede seleccionar si desea entrar como agencia o como cliente.

El cliente debe seleccionar:

> Customer / Cliente

Es importante recalcar que los clientes deben entrar por la parte de cliente, no por agencia.

## 3. Opciones iniciales dentro de la parte de cliente

Una vez dentro de la parte de cliente, el usuario verá dos opciones principales:

1. Crear demo.
2. Crear agente.

## 4. Crear demo

El demo permite configurar un agente de prueba con límite de 10 mensajes.

El objetivo del demo es que el cliente entienda cómo funciona un chatbot/agente, cómo puede darle instrucciones y qué tipo de información necesita proporcionar para que el agente responda de la manera deseada.

El demo es únicamente una prueba limitada. No representa el agente final del cliente.

## 5. Crear agente

Cuando el cliente decide crear un agente, puede elegir entre cuatro opciones:

1. MovIA Captura
2. MovIA Híbrido
3. MovIA Ventas
4. MovIA Pro Comercial

Actualmente, los productos disponibles son:

- MovIA Captura
- MovIA Híbrido

Los productos MovIA Ventas y MovIA Pro Comercial están marcados como no disponibles o futuros.

## 6. Definición del agente

Después de seleccionar el tipo de agente, el cliente debe contar qué debe hacer el agente de WhatsApp.

Se le solicitará:

- Objetivo principal.
- Meta principal.
- Descripción del agente.
- Contexto sobre el proceso que quiere automatizar o atender.

La plataforma muestra ejemplos y guías para ayudar al cliente a llenar esta información.

### Recomendación para el agente de preventa

El agente debe explicar que esta información es muy importante, porque MovIA la usará para construir el agente.

Mientras más contexto proporcione el cliente, mejores resultados podrá entregar MovIA.

## 7. Información del negocio

El cliente debe proporcionar:

- Nombre del negocio.
- Categoría del negocio.
- Ciudad.
- Nombre del contacto principal.
- WhatsApp del contacto principal.
- Correo del contacto principal.

## 8. Preferencia de contacto

El cliente debe indicar cómo prefiere ser contactado por MovIA:

- Llamada.
- Zoom.
- Mensaje de WhatsApp.
- Ambas opciones.

## 9. Revisión del pedido y depósito

Después de llenar la información inicial, el cliente revisa un resumen de su pedido.

Para iniciar el proyecto, debe pagar un depósito del 50% del valor total del proyecto.

Sin este depósito, MovIA no puede iniciar trabajo personalizado para la empresa.

El pago se realiza mediante Stripe.

MovIA no guarda información bancaria ni datos de tarjetas. Stripe procesa el pago de forma segura.

## 10. Mensualidad

Cada agente tiene una mensualidad diferente.

La mensualidad cubre:

- Mantenimiento del agente.
- Recursos computacionales.
- Ambiente de operación.
- Disponibilidad del agente.
- Ajustes dentro del alcance.
- Sesiones incluidas según el plan.

## 11. Información adicional después del depósito

Una vez que el cliente paga el depósito, se desbloquean pasos adicionales para proporcionar más contexto.

Aquí se solicita información que el agente usará para responder:

- PDFs.
- URL del negocio.
- Documentos de productos.
- Catálogos.
- Precios.
- Información del negocio.
- Preguntas frecuentes.
- Notas de negocio.

Los documentos deben ser preferentemente PDF.

## 12. Ejemplos de conversación

El cliente debe agregar ejemplos cortos de preguntas que sus clientes podrían hacer y cómo debería responder el agente.

Estos ejemplos se usan como referencia para construir y probar el agente.

Ejemplo:

- Pregunta del cliente final: “¿Cuánto cuesta el servicio?”
- Respuesta esperada: “El precio depende del tipo de servicio. Te puedo pedir tus datos para darte más información.”

Estos ejemplos ayudan a que MovIA entienda el tono, la intención y la forma esperada de respuesta.

## 13. Integración de WhatsApp

El cliente debe conectar su número de WhatsApp con MovIA.

Requisitos:

- Debe ser una cuenta de WhatsApp Business.
- No puede ser una cuenta personal.
- Debe tener una cuenta de Facebook vinculada.
- La cuenta de Facebook debe tener permisos suficientes para conectar el WhatsApp Business.

El proceso redirige al cliente a una página de conexión oficial.

MovIA utiliza la conexión oficial de Meta/Facebook para WhatsApp Business.

## 14. Revisión final de información

Después de completar los documentos, ejemplos e integración, el cliente revisa la información enviada.

Cuando el cliente completa esta etapa, MovIA recibe una notificación para iniciar la construcción del agente.

## 15. Construcción del agente

MovIA construye el agente con base en:

- Información inicial.
- PDFs.
- URL del negocio.
- Notas.
- Ejemplos de conversación.
- Tipo de agente seleccionado.
- Alcance del producto contratado.

## 16. Client Review

Cuando el agente está construido, el cliente recibe una notificación.

El cliente entra a la plataforma y revisa:

- Escenarios de prueba.
- Resultados de pruebas.
- Archivos de revisión.
- Resumen del comportamiento del agente.
- Evidencia de cómo respondió el agente en ambiente de pruebas.

El cliente puede:

1. Aprobar el agente.
2. Solicitar ajustes.

## 17. Ajustes

Si el cliente solicita ajustes, MovIA revisa los comentarios y modifica el agente dentro del alcance contratado.

No se agregan nuevas características fuera del alcance sin cotización.

## 18. Pago final

Cuando el cliente aprueba el agente, se le redirige a la pantalla para pagar el restante del proyecto.

El restante se paga únicamente después de la aprobación del agente.

## 19. Activación del agente

Una vez pagado el restante, el agente se despliega en el número de WhatsApp del cliente.

El cliente podrá ver el estado del proyecto en su plataforma:

- Pendiente.
- En construcción.
- En revisión.
- Aprobado.
- Pendiente de pago final.
- Activo.
- Pausado.
- Cancelado.

El agente no es un programa editable por el cliente. Es un agente configurado y operado por MovIA que funciona dentro de su número de WhatsApp Business.

## Mensaje corto para preventa

Si el cliente pregunta “¿cómo empiezo?”, el agente puede responder:

> Entras a app.moviatech.com.mx, te registras, eliges la opción de Cliente, seleccionas el tipo de agente que quieres crear y llenas la información inicial de tu negocio. Después revisas tu pedido y pagas el 50% de depósito para que podamos empezar a trabajar en tu agente. Una vez pagado, podrás subir documentos, ejemplos de conversación y conectar tu WhatsApp Business.
