# MovIA Híbrido: acciones externas y alcance

MovIA Híbrido incluye todas las capacidades de MovIA Captura y permite configurar hasta **dos acciones operativas acordadas** con el cliente.

## Acciones que puede incluir

Dependiendo del proceso del cliente y de la viabilidad técnica, las acciones pueden incluir:

- Agendar o administrar citas.
- Generar cotizaciones.
- Registrar información.
- Crear o actualizar un registro.
- Consultar información desde una herramienta o proceso aprobado.
- Enviar información a una herramienta o proceso aprobado.
- Enviar recordatorios.
- Dar seguimiento a leads.

Estos ejemplos no representan una lista cerrada. Las dos acciones se definen según las necesidades del cliente y el alcance acordado.

## Casos que requieren revisión técnica

Antes de confirmar una integración, MovIA debe revisar los casos que involucren:

- Un CRM específico.
- Un ERP.
- Un sistema contable.
- Una plataforma propietaria.
- Una API que todavía no haya sido revisada.
- Autenticaciones, permisos o accesos especiales.
- Creación o actualización de registros en una plataforma externa.
- Procesos con varias condiciones o reglas.
- Dudas sobre si el proceso solicitado puede resolverse dentro de las dos acciones incluidas.

En estos casos, MovIA Híbrido puede ser una opción adecuada, pero la compatibilidad y el alcance deben confirmarse después de revisar la herramienta y el proceso.

## No incluido en el paquete estándar

MovIA Híbrido no incluye:

- Más de dos acciones operativas.
- Lógica de negocio avanzada o altamente personalizada.
- Desarrollo extenso a la medida.
- Proyectos de consultoría fuera del alcance del agente.
- Integraciones ilimitadas.
- Flujos operativos ilimitados.
- Garantía de compatibilidad con una herramienta antes de realizar una revisión técnica.

## Criterio general

MovIA Híbrido sí puede realizar acciones e interactuar con herramientas o procesos externos cuando el alcance sea claro y la integración sea técnicamente viable.

No se debe afirmar que MovIA Híbrido nunca puede interactuar con sistemas externos.

Tampoco se debe garantizar una integración específica antes de revisar técnicamente la herramienta y el proceso del cliente.
